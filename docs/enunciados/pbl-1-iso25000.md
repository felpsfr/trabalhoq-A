## Problema Norteador do PBL — Projeto Local Eats

### 1. Contexto

Uma startup de tecnologia foi contratada por uma associação de comerciantes locais para desenvolver a plataforma digital **Local Eats**.

O objetivo do sistema é conectar moradores e turistas a restaurantes independentes da cidade, fortalecendo o comércio local e oferecendo uma experiência prática, rápida e confiável de busca e descoberta de estabelecimentos gastronômicos.

A plataforma possui versão web e aplicativo mobile e permite que os usuários:

- Busquem restaurantes por tipo de culinária, localização e faixa de preço
- Visualizem cardápios, fotos e avaliações
- Salvem locais favoritos
- Recebam recomendações personalizadas
- Compartilhem experiências

Para atender ao prazo de um grande evento gastronômico, a primeira versão do sistema foi desenvolvida rapidamente e colocada em produção. Após o lançamento, começaram a surgir diversos relatos de problemas por parte dos usuários.

### 2. Situação-Problema

Entre as principais reclamações registradas estão:

- O sistema fica lento em horários de pico
- Algumas telas são confusas e pouco intuitivas
- Certas buscas retornam resultados incorretos
- O aplicativo apresenta falhas em determinados modelos de smartphone
- Usuários encontram dificuldade para concluir ações simples
- Avaliações desaparecem após atualização da página
- Há inconsistências entre a versão web e a versão mobile

A associação de comerciantes está preocupada com a reputação da plataforma e precisa de uma avaliação técnica.

### 3. Desafio proposto

Vocês foram contratados como equipe de Qualidade de Software para realizar um diagnóstico inicial do produto. A direção da associação deseja respostas para as seguintes questões:

- O sistema possui qualidade adequada para continuar em operação?
- Quais aspectos da qualidade do produto estão comprometidos?
- Quais problemas devem ser priorizados?

### 4. Objetivo da atividade

Realizar uma análise preliminar da qualidade do produto de software, identificando quais atributos de qualidade podem estar comprometidos no sistema.

O foco é avaliar o produto com base em critérios técnicos de qualidade, e **não em opiniões subjetivas**.

### 5. O que vocês precisam fazer

**Passo 1 — Compreender o cenário.** Leiam atentamente o contexto e os relatos apresentados. Procurem entender:

- Qual é o propósito do sistema
- Quem são seus usuários
- Quais problemas estão sendo enfrentados

**Passo 2 — Identificar os problemas do produto.** Elaborem uma lista clara e objetiva com os problemas percebidos no sistema.

**Passo 3 — Relacionar problemas a atributos de qualidade.** Para cada problema identificado, indiquem qual característica de qualidade do produto está sendo afetada. Utilizem como base os atributos de qualidade estudados em aula (ISO/IEC 25000).

**Passo 4 — Justificar tecnicamente.** Explique de forma breve:

- Por que esse problema afeta esse atributo de qualidade
- Qual o impacto para o usuário ou para o negócio

### 6. Perguntas que ajudam na análise

- O sistema executa corretamente as funções que promete?
- É fácil de usar e entender?
- Funciona de forma estável?
- Apresenta bom desempenho?
- Funciona adequadamente em diferentes dispositivos e ambientes?
- Gera frustração ou retrabalho ao usuário?

### 7. Entrega da atividade

Cada grupo deverá entregar:

- Lista de problemas identificados no produto
- Atributos de qualidade possivelmente afetados
- Justificativas técnicas para cada relação identificada
